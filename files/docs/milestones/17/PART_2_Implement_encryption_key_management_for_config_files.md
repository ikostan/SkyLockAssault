# Milestone 17, Part #2

## Description

This PR introduces centralized encryption key management for all local
configuration files in SkyLockAssault.

It migrates settings, input mappings, audio volumes, and related persistence to
use Godot's encrypted `ConfigFile` (`load_encrypted_pass` /
`save_encrypted_pass`).


It maintains full backward compatibility with existing plaintext config files
through automatic detection and a one-time migration.

The encryption key is deterministic per-device (based on a production salt +
`OS.get_unique_id()`), while supporting overrides for test environments and
web builds.

## Changes

- [x] **Core Architecture & Security**: Added `save_encryption_pass` and
  `ensure_encryption_key()` to `scripts/core/globals.gd`. Derives a SHA-256 key
  from device ID and `ProjectSettings` salt with production guard rails.
 
- [x] **Component Migrations**: Updated `settings.gd` and `audio_manager.gd` to
  use centralized safe loading + encrypted saving. Preserves legacy support,
  migration flags, deduplication, and conflict resolution.
 
- [x] **Data Preservation**: Ensured multi-writer safety. Configs are pre-loaded
  before saving to preserve unrelated sections (e.g., preventing settings
  updates from wiping audio configs).
 
- [x] **CI/CD & Deployment**: Added `PRODUCTION_SALT` GitHub secret support.
  Securely injects the salt into `project.godot` under the `[game]` section
  during Itch.io builds using section-aware `awk` scripting.
 
- [x] No breaking changes for players; existing plaintext saves migrate
  seamlessly to encrypted formats on first load/save.

  read/write using the new encrypted persistence approach. Added dedicated
  suite `test_settings_migration.gd` to cover new-install encryption, legacy
  plaintext fallback, and lossless multi-writer migration.
 
- [x] **E2E Testing (Playwright)**: Updated Python Playwright flows. Relaxed
  timeouts and added web-specific initialization flags
  (`window.godotInitialized`) for more reliable browser-based UI/audio testing.
 
- [x] **Manual Testing**: Verified safe coexistence and multi-writer safety
  between `settings.gd` and `globals.gd` on the shared `user://settings.cfg`
  file. Unrelated sections are preserved perfectly during loads/saves.
